前言

三方库：Selenium
什么是自动化：
自动化是指使用技术手段模拟人工，执行重复性任务。准确性100%

自动化应用场景：

  1. 自动化测试
  2. 自动化运维
  3. 自动化办公
  4. 自动化游戏

Selenium

  • web自动化中的老牌指名开源库
  • 支持多种浏览器：谷歌Chrome、火狐Firefox、微软Edge、苹果Safari等
  • 跨平台、兼容性高
  • 通过浏览器驱动控制浏览器，通过元素定位模拟人工交互以实现web自动化（无焦点状态依然执行）

官方技术文档
https://www.selenium.dev/

本笔记内容：

  • Selenium库的所有常用语句和功能
  • 浏览器配置、元素定位、元素交互
  • 多种web场景下的案例、多线程执行（多开）等等

Selenium环境安装

  1. 谷歌Chrome浏览器右上角…-帮助-关于，确认浏览器版本
  2. 安装浏览器驱动，驱动的版本号要和浏览器的版本号对应符合（至少大版本号要一致）。
    官方最新驱动下载地址：https://googlechromelabs.github.io/chrome-for-testing/
  3. 把驱动的exe文件放在项目的根目录下
  4. 安装三方库，在终端中执行 在后面加’==3.141.0’（版本号）就是指定版本下载

元素定位-ID

  1. 通过ID定位元素，一般比较准确。
  2. 并不是所有网页或者元素都有ID值

元素定位-NAME

  1. 通过NAME定位元素，一般比较准确。
  2. 并不是所有网页或者元素都有NAME值

元素定位-CLASS_NAME

  1. class值不能有空格，否则报错
  2. class值重复的有很多，需要切片
  3. class值有的网站是随机的，每次刷新都会改变

元素定位-TAG_NAME

  1. 查找<开头标签名字>
  2. 重复的标签名字特别多，需要切片

元素定位-LINK_TEXT

  1. 通过精准链接文本找到标签a的元素
  2. 有重复的文本需要切片

元素定位-PARTIAL_LINK_TEXT

  1. 通过模糊链接文本找到标签a的元素[模糊文本定位]
  2. 有重复的文本，需要切片

元素定位-CSS_SELECTOR

  1. #id = 井号+id值 通过id定位

  2. .class = 点+class值 通过class定位

  3. 不加修饰符 = 标签头 通过标签头定位

  4. 通过任意类型定位："[类型 = '精准值']"

  5. 通过任意类型定位："[类型* = '模糊值']"

  6. 通过任意类型定位："[类型^ = '开头值']"

  7. 通过任意类型定位："[类型$'结尾值']"
    以上这些方法都属于理论定位法

  8. 更简单的定位方式：在谷歌控制台直接复制SELECTOR （个别元素的定位值会比较长）

元素定位-XPATH

  1. 复制谷歌浏览器 Xpath （通过属性+路径定位，属性如果是随机的，可能定位不到）
  2. 复制谷歌浏览器 Xpath 完整路径 （缺点是定位值比较长，优点是基本100%精确）

元素定位隐形等待

多少秒内找到元素就立刻执行，没找到元素就报错

单选、多选、下拉元素交互

单选：定位元素复制Xpath即可
多选：定位元素依次复制Xpath即可
下拉：定位元素复制Xpath即可（有些时候可以直接定位，有些时候需要点击开下拉列表后再定位）

日期、评星、上传元素交互

日期：定位元素复制Xpath，使用send_keys直接写入
评星：定位元素复制Xpath即可
上传：定位元素复制Xpath，使用send_keys写入上传文件的全部绝对路径
